.NET Core Web API

Content Negotiation

The HTTP mechanism that lets the same endpoint return JSON, XML, CSV or any format — based on what the client asks for. Zero extra code per format.

🤝

The Handshake

The client tells the server "I can understand these formats" via the Accept header. The server picks the best match it supports and responds accordingly.

RFC 7231 §5.3
📨

Accept Header

Client sends:
Accept: application/json, application/xml;q=0.8, */*;q=0.5
The q value is the quality factor — priority 0 to 1.

Request header
📤

Content-Type Header

Server responds with the actual format it chose:
Content-Type: application/json; charset=utf-8

Response header
⚡

Why it matters

One /api/products endpoint serves a React app (JSON), a legacy Java system (XML), and an Excel export (CSV) — without duplication.

Single endpoint
🚫

406 Not Acceptable

If the server can't produce any of the client's requested formats, it returns HTTP 406 Not Acceptable — the content negotiation failed.

HTTP 406
🔧

In .NET Core

ASP.NET Core has built-in content negotiation. Just register formatters in Program.cs and return Ok(data) — the framework handles the rest.

Zero boilerplate

Same Data, Different Formats
One GET /api/products/1 endpoint — different Accept headers, different responses

application/json

  • Default in .NET Core
  • Most web frontends
  • Compact & readable
  • Built-in via System.Text.Json

application/xml

  • Legacy enterprise systems
  • SOAP integrations
  • Add Newtonsoft XML package
  • Verbose but self-describing

text/csv

  • Excel / spreadsheet exports
  • Data pipeline ingestion
  • Custom formatter needed
  • Lightest payload size

406 Not Acceptable

  • No matching formatter found
  • Client asked for unsupported type
  • RespectBrowserAcceptHeader flag
  • Fallback: default formatter

.NET Core Setup — Step by Step
Everything you need to enable content negotiation in your Web API project
1
Register formatters in Program.cs
JSON is built-in. Add XML via AddXmlSerializerFormatters(). CSV needs a custom formatter.
var builder = WebApplication.CreateBuilder(args); builder.Services.AddControllers(options => { // Return 406 if no formatter matches (recommended) options.ReturnHttpNotAcceptable = true; // Respect the browser's Accept header options.RespectBrowserAcceptHeader = true; }) .AddXmlSerializerFormatters() // adds application/xml .AddXmlDataContractSerializerFormatters(); // optional: DataContract XML
2
Install NuGet package for XML
The XML formatter is in a separate package. JSON works out of the box.
$ dotnet add package Microsoft.AspNetCore.Mvc.Formatters.Xml # Or via Package Manager Console: PM> Install-Package Microsoft.AspNetCore.Mvc.Formatters.Xml
3
Create your API controller — no format code needed!
Return Ok(data) and .NET Core handles the rest based on the Accept header.
[ApiController] [Route("api/[controller]")] public class ProductsController : ControllerBase { [HttpGet("{id}")] public IActionResult GetProduct(int id) { var product = new Product { Id = id, Name = "Mechanical Keyboard", Price = 129.99m, InStock = true }; // ASP.NET Core picks JSON or XML automatically // based on the client's Accept header! return Ok(product); } }
4
Custom CSV formatter (bonus)
For unsupported formats, implement OutputFormatter. Here's a minimal CSV example.
public class CsvOutputFormatter : TextOutputFormatter { public CsvOutputFormatter() { SupportedMediaTypes.Add("text/csv"); SupportedEncodings.Add(Encoding.UTF8); } protected override bool CanWriteType(Type? type) => typeof(Product).IsAssignableFrom(type); public override async Task WriteResponseBodyAsync( OutputFormatterWriteContext context, Encoding selectedEncoding) { var product = (Product)context.Object!; var csv = $"Id,Name,Price,InStock\n{product.Id},{product.Name},{product.Price},{product.InStock}"; await context.HttpContext.Response.WriteAsync(csv, selectedEncoding); } } // Register in Program.cs: options.OutputFormatters.Insert(0, new CsvOutputFormatter());
5
Test with curl or Postman
Change the Accept header to get different formats from the same endpoint.
# Request JSON (default) $ curl -H "Accept: application/json" https://localhost:5001/api/products/1 # Request XML $ curl -H "Accept: application/xml" https://localhost:5001/api/products/1 # Request CSV $ curl -H "Accept: text/csv" https://localhost:5001/api/products/1 # Trigger 406 (unsupported format) $ curl -H "Accept: application/pdf" https://localhost:5001/api/products/1 # → HTTP 406 Not Acceptable
